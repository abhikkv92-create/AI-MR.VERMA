# Trigger: /penetration-tester

Use this command to specifically engage the **penetration-tester** for tasks within their domain.

1. **Context Analysis**
   - Identify the specific penetration-tester requirements in the user request.
   - Cross-reference with `@[agent-skills/penetration-tester]` if applicable.

2. **Engage Agent**
   - Use the **penetration-tester** agent to process the task.
   - Enforce symbolic density per `@[/poweruseage]`.
   - Perform memory profiling per `@[/memory-optimization]`.

3. **Interconnection**
   - Proactively suggest related workflows (e.g., if debugging, suggest `/audit`).
