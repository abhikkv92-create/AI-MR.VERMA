# Trigger: /mobile-developer

Use this command to specifically engage the **mobile-developer** for tasks within their domain.

1. **Context Analysis**
   - Identify the specific mobile-developer requirements in the user request.
   - Cross-reference with `@[agent-skills/mobile-developer]` if applicable.

2. **Engage Agent**
   - Use the **mobile-developer** agent to process the task.
   - Enforce symbolic density per `@[/poweruseage]`.
   - Perform memory profiling per `@[/memory-optimization]`.

3. **Interconnection**
   - Proactively suggest related workflows (e.g., if debugging, suggest `/audit`).
